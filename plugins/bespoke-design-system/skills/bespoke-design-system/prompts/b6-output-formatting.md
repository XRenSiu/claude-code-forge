# B6 — 输出与协商接口

> 当 SKILL.md 主流程进入 B6 时读取本文件作为执行指引。

## 你的任务

把 B4/B5 通过的草稿落盘为 3 份产物，触发自演化机制（更新 adaptation-stats），给用户一个清晰的协商接口。

---

## Step 1 — 决定产物路径

默认路径：`./bespoke-design/<slug>/`

- `<slug>` 从 product_brief 提取，kebab-case，例如 "八字 SaaS" → `bazi-saas`，"AI 写作助手" → `ai-writing-assistant`
- 若当前目录就是产品仓库（含 `package.json` / `pyproject.toml` / `go.mod` 等），询问用户偏好：
  - 选项 A：DESIGN.md 写到 `./DESIGN.md`，provenance/negotiation 放到 `./bespoke-design/`
  - 选项 B：所有产物都进 `./bespoke-design/<slug>/`
  - 默认 A
- 若用户在调用时显式给了 `--output <path>`，按指定写

**注意**：这是 B6，**不算追问用户信息**。这里问的是产物路径偏好，不影响生成本身。但能省略就省略——若有强信号（用户说"放到项目根"），直接执行不问。

## Step 2 — 写 DESIGN.md（OD 方言 9-section）

按 OD 标准格式输出。每个 section 用 `## <数字>. <Section Name>` 二级标题（数字编号是 OD 约定）。

**模板**（详见 `references/design-md-spec.md`）：

```markdown
# Design System for <Product Name>

> Generated by bespoke-design-system v1.0.0 on <date>
> Mode: interactive | auto
> Source systems referenced: <list>

## 1. Visual Theme & Atmosphere
<3-4 段散文 + Key Characteristics 列表>

## 2. Color Palette & Roles
<按 role 分组的色板 + Strategy 子段>

## 3. Typography Rules
<Font Family + Hierarchy 表 + Principles>

## 4. Component Stylings
<Buttons / Cards / Inputs / Navigation / Distinctive>

## 5. Layout Principles
<Spacing / Grid / Whitespace Philosophy / Border Radius>

## 6. Depth & Elevation
<层级表 + Shadow Philosophy 散文>

## 7. Do's and Don'ts
### Do
- ...
### Don't
- ...

## 8. Responsive Behavior
<Breakpoints 表 + Touch Targets + Collapsing Strategy + Image Behavior>

## 9. Agent Prompt Guide
### Quick Color Reference
- ...
### Example Component Prompts
- ...
### Iteration Guide
1. ...
```

**所有 9 个 section 必须存在**且**按上述顺序、含数字编号、严格使用 OD 方言名字**。某个 section 在 B3 被标 `degraded` → 写 default 但在 section 顶部加注 `> ⚠️ degraded section: rules library coverage was insufficient for this section. Defaults applied.`

**绝不**：
- 把 section 重命名（如 "Color" / "Anti-Patterns" / "Voice" / "Motion" / "Brand"——这是非标准命名，OD 工具不消费）
- 漏掉数字编号
- 改变顺序
- 新增第 10 个顶级 section（扩展用 Appendix）

## Step 3 — 写 provenance.yaml

把 B4 输出的 Provenance Report（每个决策的三段式）合并为一个 YAML 数组，加上头部元数据：

```yaml

  generator: bespoke-design-system
  version: 1.0.0
  generated_at: <ISO 8601>
  user_profile_hash: <sha256(profile)>
  rules_library_version: <从 grammar/meta/version.json>
  source_systems_referenced: [linear, vercel, ...]
  mode: interactive | auto
  gate_rounds: 1 | 2
  inferred_fields: [...]   # auto 模式才有

decisions:
  - decision: ...
    section: ...
    inheritance: {...}
    adaptation: {...}
    justification: {...}
    confidence: high | medium | low

dropped_during_b3:
  - rule_id: ...
    reason: ...

degraded_sections: [...]
```

## Step 4 — 写 negotiation-summary.md

这是给**用户**读的，不是给系统的。语言要自然、克制、有立场。

模板：

```markdown
# Negotiation Summary — <Product Name>

## 这份设计的核心调性

<3-5 句自然语言。基于 B1 画像的 archetype + kansei 写出来。例如：
"这是一份属于 Sage+Magician 的设计——既要让用户感到这里有古老智慧的可信度（structured / ancient / mystical），
又不能让它看起来像封建迷信（reverse_constraint: 算命店感）。
整体调性偏冷、偏暗、偏少装饰，靠字体的克制和空间的呼吸感传达神秘。"
>

## 关键决策

| 决策 | 一句话 rationale | 来源系统 |
|---|---|---|
| 主色 #4A4080 (深紫蓝) | Linear 的"严肃但不冷漠"基调，调暗 + 降饱和度增加古老感 | linear, vercel |
| 字体：Cormorant Garamond + Inter | 衬线显示 + 无衬线正文：传统厚重的标题感 + 现代克制的可读性 | new-york-times, stripe |
| Spacing 1.5× base | 神秘感需要呼吸感，比常规 SaaS 的 1.0× 更松 | apple, notion |
| ... | ... | ... |

## 推断字段（仅 auto 模式）

> 这份设计在 auto 模式下生成。以下字段是基于上下文推断的——如果不准，告诉我具体哪一条，我会重排规则重新生成。

- `audience`: 推断为 "25-45 岁对传统命理感兴趣的现代用户"（来源：产品名"八字 SaaS"+ 默认 spiritual_saas 受众）
- `competitive_reference`: 推断为"避免淘宝算命店感"（来源：reverse_constraint default for spiritual_saas）
- ...

## 协商接口

如果你想改某个决策：

- 例 1："我不要 ancient，改成 minimalist" → 我会回到 B3 重排，去掉 ancient 相关规则，加入 minimalist 主导的规则
- 例 2："主色再亮一点" → 我会回到 B4 调整 lightness 维度的 adaptation
- 例 3："整体偏冷我接受不了" → 我会回到 B1 修改 SD warm_to_cold 然后重跑 B2-B6

告诉我具体哪个决策、改成什么方向。

## 已知限制

- <列出 B5 闸门给的非 blocker warnings>
- <列出 degraded sections>
- <列出 anti-slop check 的 suspicions>
```

## Step 5 — 触发自演化

读 `grammar/meta/adaptation-stats.json`（不存在则初始化为 `{ adaptations: {} }`）。

对 B4 产生的每条 `adaptation.modifications`，生成稳定的 `adaptation_id`：

```
adaptation_id = <source_rule_id>:<dimension>_<direction>_for_<top_kansei_match>
```

例如 `linear-color-balance-001:saturation_decrease_for_mystical`。

更新条目：

```json
{
  "adaptation_id": "linear-color-balance-001:saturation_decrease_for_mystical",
  "occurrences": 7,
  "contexts": [
    {
      "user_kansei": ["mystical", "ancient"],
      "from": 0.45,
      "to": 0.30,
      "generated_at": "2026-05-03T..."
    }
  ],
  "last_seen": "2026-05-03"
}
```

写回 `grammar/meta/adaptation-stats.json`。

**沉淀提示**：扫描 stats 中 `occurrences ≥ 5` 的 adaptation。如果有：

```
💡 检测到 N 条高频 adaptation 已达沉淀阈值（occurrences ≥ 5）：
   - linear-color-balance-001:saturation_decrease_for_mystical (7 次)
   - ...
   你可以运行"沉淀 adaptation"维护命令，把它们提议为新规则加入规则库。
```
