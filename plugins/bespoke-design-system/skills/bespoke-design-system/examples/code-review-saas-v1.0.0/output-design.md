# Design System for Code Review SaaS

> Category: Developer Tools
> Lightweight code review focused on judgment-worthy diffs. Engineered restraint, not enterprise SaaS template.
> Generated by bespoke-design-system v1.0.0 on 2026-05-03 (mode: auto)
> Source systems referenced: linear-app, vercel

## 1. Visual Theme & Atmosphere

This design system inhabits a dark-native medium — a near-black canvas (`#0c0d0f`) where review interfaces emerge from controlled luminance rather than vibrant color. Every UI surface is calibrated for one purpose: keep the reviewer's eye on the diff, not on the chrome around it. The palette is achromatic except for a single restrained chromatic anchor (a calmed teal-leaning blue, `#4d7df0`) reserved exclusively for primary CTAs and active state markers. There are no decorative gradients, no marketing-grade illustrations — every visual choice has a structural justification, like code that's been reviewed for redundancy.

Typography is the loudest voice in this system, and even it whispers. Inter Variable with `cv01, ss03` OpenType features carries the entire interface, with the signature 510 weight as the primary emphasis (between regular and medium — a between-weight that suggests rather than declares). Display sizes use aggressive negative letter-spacing (-1.05px at 48px, -0.7px at 32px) so headlines read as compressed, engineered, urgent. The narrow weight palette (400 / 510 / 590, no bold) creates hierarchy through size and tracking rather than weight, mirroring how good diffs use line position rather than color to communicate change.

**Key Characteristics:**
- Dark-mode-native with luminance stepping (`#0c0d0f` canvas → `#161719` panel → `#1f2023` elevated)
- Single chromatic anchor: calmed teal-blue `#4d7df0` (CTA + active accent only)
- Inter Variable with `"cv01", "ss03"` globally; signature weight 510
- Aggressive negative letter-spacing at display, relaxing to normal at body
- Semi-transparent white borders (`rgba(255,255,255,0.05–0.08)`) — never solid dark
- Voice: engineered restraint, not orchestral marketing
- 8px base spacing with optical micro-corrections (4 / 7 / 8 / 11 / 12 / 16 / 24 / 32)

## 2. Color Palette & Roles

### Background Surfaces
- **Canvas Black** (`#0c0d0f`): The deepest background — page canvas for review queue, dashboards, marketing pages.
- **Panel Dark** (`#161719`): Sidebars, side panels, secondary navigation regions.
- **Elevated Surface** (`#1f2023`): Cards, dropdowns, dialog headers — one luminance step above panel.
- **Hover Surface** (`#26272b`): Hover state on elevated surfaces.

### Text & Content
- **Primary Text** (`#f4f5f7`): Default text color — near-white but warm enough to avoid eye strain.
- **Secondary Text** (`#cbd0d9`): Body copy, descriptions.
- **Tertiary Text** (`#878d97`): Metadata, timestamps, line numbers in diffs.
- **Quaternary Text** (`#5d626c`): Disabled states, deeply de-emphasized labels.

### Brand & Accent (Single Chromatic)
- **Review Blue** (`#4d7df0`): The only chromatic in the system — used for primary CTA, active selection of "current diff", commit hash links.
- **Review Blue Hover** (`#688fff`): Lighter variant for hover.
- **Review Blue Subtle** (`rgba(77,125,240,0.12)`): Very low-alpha for selected-row backgrounds in the diff list.

### Status Colors (Functional Only)
- **Reviewer Approved** (`#27a644`): "Approved" badge.
- **Changes Requested** (`#e25c5c`): "Changes requested" badge.
- **Pending** (`#dba23b`): "Awaiting review" badge.
- These status colors are used **only** in their semantic context — never decoratively.

### Border & Divider
- **Border Subtle** (`rgba(255,255,255,0.05)`): Default border for cards and panels.
- **Border Standard** (`rgba(255,255,255,0.08)`): Stronger border for inputs, dialog containers.
- **Border Strong** (`rgba(255,255,255,0.12)`): Reserved for critical separations (e.g., diff sidebar / diff content).

### Strategy
- **Palette type**: monochromatic dark with single chromatic accent
- **Contrast strategy**: luminance stepping; primary text 16:1 contrast on canvas, secondary 9:1, tertiary 5:1
- **Body-on-bg contrast ratio**: 16:1 (`#f4f5f7` on `#0c0d0f`) — exceeds WCAG AAA

## 3. Typography Rules

### Font Family
- **Primary**: `Inter Variable`, with fallbacks `SF Pro Display, -apple-system, system-ui, Segoe UI`
- **Monospace**: `JetBrains Mono`, with fallbacks `Berkeley Mono, ui-monospace, SF Mono, Menlo`
- **OpenType Features**: `"cv01", "ss03"` enabled globally. `"tnum"` enabled on diff line numbers and metric displays.

### Hierarchy

| Role | Font | Size | Weight | Line Height | Letter Spacing | Notes |
|------|------|------|--------|-------------|----------------|-------|
| Display | Inter Variable | 48px (3.00rem) | 510 | 1.00 | -1.056px | Marketing hero, empty state title |
| Heading 1 | Inter Variable | 32px (2.00rem) | 510 | 1.13 | -0.704px | Section titles |
| Heading 2 | Inter Variable | 24px (1.50rem) | 510 | 1.33 | -0.288px | Sub-section, card title |
| Heading 3 | Inter Variable | 18px (1.13rem) | 590 | 1.40 | -0.165px | Diff group title |
| Body | Inter Variable | 14px (0.88rem) | 400 | 1.50 | normal | Standard reading text |
| Body Emphasis | Inter Variable | 14px (0.88rem) | 510 | 1.50 | normal | UI labels, navigation |
| Body Strong | Inter Variable | 14px (0.88rem) | 590 | 1.50 | normal | Active state, key labels |
| Caption | Inter Variable | 12px (0.75rem) | 510 | 1.40 | normal | Metadata, timestamps |
| Diff Code | JetBrains Mono | 13px | 400 | 1.50 | normal | Diff content (line of code) |
| Line Number | JetBrains Mono | 12px | 400 (tnum) | 1.50 | normal | Tabular line numbers |
| Status Label | Inter Variable | 11px (0.69rem) | 590 | 1.40 | 0.04em | Uppercase status badges |

### Principles
- **510 is the workhorse**: Use Inter Variable's 510 weight for all UI labels, navigation, and emphasis. Reserve 590 for active state and strong contrast moments.
- **Compression at scale**: Display headlines run with -1.056px tracking at 48px. Tracking relaxes to normal at body sizes (14px and below).
- **OpenType features are non-negotiable**: `"cv01", "ss03"` must be active on every Inter element. Without them, Inter is generic — with them, it has identity.
- **Mono for code only**: JetBrains Mono renders all diff code, commit hashes, and tabular numbers. Never use mono for body text or labels.

## 4. Component Stylings

### Buttons

**Primary CTA**
- Background: `#4d7df0`
- Text: `#ffffff`
- Padding: 8px 16px
- Radius: 6px
- Hover: `#688fff`
- Use: "Submit review", "Approve", "Request changes" — only the single most important action per view

**Ghost (Default)**
- Background: `rgba(255,255,255,0.02)`
- Text: `#cbd0d9`
- Padding: 6px 12px
- Radius: 6px
- Border: `1px solid rgba(255,255,255,0.08)`
- Hover: bg shifts to `rgba(255,255,255,0.05)`
- Use: Secondary actions, "Skip", "Later"

**Subtle**
- Background: `rgba(255,255,255,0.04)`
- Text: `#878d97`
- Padding: 0px 6px
- Radius: 4px
- Use: Toolbar actions, line-level inline actions in diff

**Pill (Filter / Status)**
- Background: transparent
- Text: `#cbd0d9`
- Padding: 0px 10px 0px 5px
- Radius: 9999px (pill — only for filters/status, never for primary actions)
- Border: `1px solid rgba(255,255,255,0.08)`
- Font: 12px weight 510

### Cards & Containers
- Background: `rgba(255,255,255,0.02–0.04)` (always translucent — no solid surface backgrounds)
- Border: `1px solid rgba(255,255,255,0.05–0.08)` (semi-transparent white, not solid dark)
- Radius: 8px (standard), 12px (featured / diff container), 6px (compact)
- Hover: subtle background opacity increase to `0.05–0.06`
- No drop shadows on dark canvas — depth via luminance stepping

### Inputs & Forms
- **Text input**: `rgba(255,255,255,0.02)` bg, `1px solid rgba(255,255,255,0.08)` border, 6px radius, 12px 14px padding, `#f4f5f7` text
- **Comment textarea**: same as text input but min-height 80px, expands on content
- **Search input**: transparent bg, full width, command palette pattern (`Cmd+K`)
- **Validation states**: error border shifts to `rgba(226,92,92,0.4)`; success to `rgba(39,166,68,0.4)` — never solid red/green

### Cards in the Diff View
- **Diff line container**: row-based, 1.5 line-height for code, hover row gets `rgba(255,255,255,0.03)` background
- **Comment thread card**: indented under the line being commented on, `rgba(255,255,255,0.04)` bg, `rgba(77,125,240,0.3)` left-border accent (the only chromatic use in diff body)
- **Reviewer summary card**: elevated surface (`#1f2023`), 12px radius

### Navigation
- Top sticky header on `#0c0d0f`
- Logomark left-aligned (15px height)
- Links: Inter Variable 14px weight 510, `#cbd0d9` text
- Active state: text lightens to `#f4f5f7` + 1px bottom underline at `#4d7df0`
- Right-side: command palette trigger (`/` or `Cmd+K`) + user avatar
- Mobile: collapses behind hamburger at < 768px

## 5. Layout Principles

### Spacing System
- **Base unit**: 8px
- **Scale**: 4, 7, 8, 11, 12, 16, 20, 24, 32, 48, 64, 96
- **Optical adjustments**: 7px and 11px reserved for specific optical alignment (e.g., aligning diff line numbers to text baseline)
- **Primary rhythm**: 8 / 16 / 24 / 32 (standard 8px grid)
- **Section padding**: minimum 64px vertical between major page regions; 96px+ on landing/marketing pages

### Grid & Container
- **Max content width**: 1280px (review interface) — wider than typical SaaS to give diffs room
- **Diff page layout**: 240px sidebar (file tree) + flex content (diff) — no max-width on diff content; let it use all available width
- **Marketing/dashboard**: centered single-column with 1200px max-width

### Whitespace Philosophy
- **Darkness as space**: The dark canvas IS the negative space. Empty regions are intentional, not unfinished.
- **Compressed text, expanded surroundings**: Headlines compress with negative tracking; surrounding padding stays generous. The contrast between density and spaciousness creates focus.
- **No decorative dividers**: Section boundaries communicated through vertical padding (≥ 64px), not horizontal lines. Lines reserved for diff structure (file headers, hunk separators).

### Border Radius Scale
- **Micro (2px)**: Inline status pills in diff lines
- **Small (4px)**: Compact buttons, subtle tags
- **Standard (6px)**: Buttons, inputs, links
- **Card (8px)**: Cards, dropdowns, popovers
- **Featured (12px)**: Diff container, summary panels
- **Pill (9999px)**: Filter chips, status badges only

### Breakpoints
| Name | Width | Key Changes |
|------|-------|-------------|
| Mobile | < 600px | Single column, file tree as drawer |
| Tablet | 600–1024px | File tree collapsible |
| Desktop | 1024–1400px | Full layout, sticky sidebar |
| Wide | > 1400px | Diff content uses full width up to 1600px |

## 6. Depth & Elevation

| Level | Treatment | Use |
|-------|-----------|-----|
| Flat (Level 0) | No shadow, `#0c0d0f` bg | Page canvas, deepest surface |
| Surface (Level 1) | `rgba(255,255,255,0.02)` bg + `1px solid rgba(255,255,255,0.05)` | Subtle containers, nested cards |
| Standard (Level 2) | `rgba(255,255,255,0.04)` bg + `1px solid rgba(255,255,255,0.08)` | Cards, comment threads, dropdowns |
| Elevated (Level 3) | `#1f2023` solid bg + `1px solid rgba(255,255,255,0.08)` + `inset 0 0 0 1px rgba(255,255,255,0.04)` | Reviewer summary, dialog headers |
| Modal (Level 4) | `#1f2023` bg + multi-layer inset glow + `rgba(0,0,0,0.85)` backdrop | Modals, command palette |
| Focus | `0 0 0 2px rgba(77,125,240,0.4)` ring | Keyboard focus on interactive elements |

**Shadow Philosophy**: This system uses background luminance stepping rather than drop shadows for depth, because drop shadows on dark surfaces are nearly invisible. Each elevation level corresponds to a small increase in white opacity on the surface background. The only "shadow" in the system is the focus ring (using the brand chromatic at low alpha).

## 7. Do's and Don'ts

### Do
- Build on near-black canvas (`#0c0d0f`); use luminance stepping (`#161719` → `#1f2023`) for elevation
- Use semi-transparent white borders (`rgba(255,255,255,0.05–0.12)`) instead of solid dark borders
- Use Inter Variable with `"cv01", "ss03"` on all text — non-negotiable
- Reserve `#4d7df0` (Review Blue) exclusively for primary CTAs and active state markers
- Apply aggressive negative letter-spacing at display sizes (-1.056px at 48px)
- Use weight 510 as your default emphasis weight; 400 for body, 590 for strong emphasis
- Keep button backgrounds nearly transparent (`rgba(255,255,255,0.02–0.05)`) for ghost variants
- Use `#f4f5f7` for primary text — not pure `#ffffff`
- Use status colors (`#27a644` / `#e25c5c` / `#dba23b`) only in their semantic context

### Don't
- Don't use pure white (`#ffffff`) as primary text — `#f4f5f7` prevents eye strain
- Don't use solid colored backgrounds for ghost / secondary buttons — transparency is the system
- Don't apply Review Blue decoratively — it's the single chromatic anchor and must remain rare
- Don't use positive letter-spacing on display text
- Don't use opaque borders on dark backgrounds
- Don't skip OpenType features (`"cv01", "ss03"`) on Inter — without them, it's generic Inter, not this system's Inter
- Don't use weight 700 (bold) — maximum is 590
- Don't introduce warm colors into UI chrome — palette is cool gray + cool chromatic anchor only
- Don't use drop shadows for elevation on dark surfaces — luminance stepping is the system
- Don't use a CSS border on cards when a `1px` semi-transparent overlay would do — keep border weight whisper-level
- Don't use pill radius (9999px) on the primary CTA button — pills are for filter/status only

## 8. Responsive Behavior

### Touch Targets
- Buttons: minimum 36px height with comfortable padding
- Diff line tap targets: full row width, 24px minimum row height
- Pill tags: 10px horizontal padding for tap accessibility

### Collapsing Strategy
- Hero text: 48px → 32px on tablet, 24px on mobile, tracking adjusts proportionally
- Navigation: horizontal links + CTAs → hamburger at < 768px
- File tree (in diff view): persistent sidebar at desktop → collapsible drawer at tablet/mobile
- Comment threads: side-by-side at desktop → stacked below diff line at mobile
- Section spacing: 64px+ → 32px on mobile

### Image Behavior
- Marketing screenshots maintain border treatment at all sizes
- Avatar sizes scale: 24px (compact lists) → 32px (default) → 40px (profile context)

### Iteration Guide
1. Set `font-feature-settings: "cv01", "ss03"` on every Inter element — non-negotiable for system identity
2. Three Inter weights only: 400 (body), 510 (emphasis/UI), 590 (strong/active). No bold (700)
3. Reserve `#4d7df0` for primary CTA + active state — every other use of chromatic is wrong
4. Surface elevation = background opacity step, not drop shadow
5. Borders are always semi-transparent white, never solid dark
6. JetBrains Mono for code/numbers/hashes; Inter for everything else
7. Status colors (green/red/amber) are functional, never decorative
