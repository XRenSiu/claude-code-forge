## Subject Type

`self` distills the **user themselves** into a skill: a reflective surface the user can consult to check "what would I normally do here, given my known patterns and memory?" Not a therapist, not an assistant вЂ” a mirror.

Subject type in manifest: `identity.subject_type = "real-person"` and the subject is the same entity as the user running `distill-meta`. This is the ONLY schema where the user has full access to their own corpus.

## Required Components

- `hard-rules` вЂ” Layer-0 guardrails (what this skill refuses to role-play, e.g., impersonating other real people in the user's life).
- `identity` вЂ” one-paragraph self-card.
- `self-memory` вЂ” chronological + thematic memory library derived from diaries / chats with others. The distinctive component of this schema.
- `persona-5layer` вЂ” 5-layer reaction model (surface speech в†’ interaction style в†’ values в†’ drivers в†’ hidden contradictions).
- `expression-dna` вЂ” 7-axis vocal fingerprint of how the user writes/speaks.
- `honest-boundaries` вЂ” в‰Ґ3 explicit admissions of what this self-mirror does NOT know about the user (blind spots).
- `correction-layer` вЂ” accumulates user corrections ("actually I wouldn't say that") over time.

## Optional Components

- `computation-layer` вЂ” only when the user has a personal calculation ritual to preserve (custom decision spreadsheet, macro collection). Rare.

## Runtime Execution Logic

Per PRD В§7 Schema 1:

```
query received
  в†’ persona-5layer decides the reaction mode (is this a values question? a craft question? an emotional one?)
  в†’ self-memory retrieves episode(s) matching the situation (e.g., "the last time I faced a similar choice")
  в†’ expression-dna shapes the answer in the user's voice
  в†’ honest-boundaries guards against fabricating memory the user never had
```

Key ordering rule: **persona-5layer runs before self-memory** вЂ” the persona decides whether memory is even relevant before expensive retrieval.

## Quality Gate Hints

The `self` schema is especially sensitive to these `persona-judge` rubric dimensions:

- **Voice Fidelity** (expression-dna axes) вЂ” does this sound like the user, not a generic "reflective voice"?
- **Memory Groundedness** вЂ” are retrieved episodes real or fabricated? `persona-judge` boundary test must catch hallucinated childhood memories.
- **Self-Honesty** вЂ” does the persona admit blind spots or narrate an idealized self? Check `honest-boundaries`.
- **Contradiction Preservation** вЂ” does Layer 5 of `persona-5layer` surface at least one real internal tension?

## Unvalidated Caveats

- No dog-food `self` persona has been generated by this pipeline. Shipping with `unvalidated: true`.
- `self-memory` в†” `persona-5layer` ordering is theoretical; real trials may show memory-first works better for nostalgic queries.
- The redaction policy for private chat corpus is drafted but not battle-tested; expect PII leaks until Phase 4 hardening.
- There is a **psychological risk** the author hasn't explored: a well-trained `self` mirror may tell users things they don't want to hear. No safeguard is designed for this yet.
