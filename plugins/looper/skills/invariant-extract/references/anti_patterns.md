## 三个自带的失败模式

### ① 溯因过度泛化 —— 两个正交轴都会跑偏(溯因通道)

一个 obstacle 的取反是欠定的:候选不是一条,而是**一整族规则**,沿两个**正交**的轴散开。
两个轴各有一个旋钮,缺一个都救不了:

- **scope(宽窄)轴 → 奥卡姆旋钮。** 从一次超时,你可能溯出"永不调该 API",真正该写的是"永不在
  没有重试预算下调该 API"。→ 取覆盖该 obstacle 的**最窄**规则。
- **aspect(维度)轴 → 目的旋钮。** 同一超时,在正确性领地与时延领地溯出的是两条**不同维度**的 □。
  → **先按目的把 obstacle 投到正确的 aspect 子空间**(投影 ∅ 则跳过),奥卡姆才在子空间里收窄。

→ **对策**:**投影(目的)定维度 → 取反 → 奥卡姆(最窄)收宽窄 → 绑死到那个具体 obstacle**。
**沿错误维度取的最窄规则照样错**;只调奥卡姆、不调目的,是这条失败模式最隐蔽的半个。
详见 `references/abduction.md` 二点五 + 三。

### ② 低置信溯因自动入卡(溯因通道 / 落卡阶段)

溯因不是演绎。宽的、高风险的候选,skill 只**提案**,触发 `NEEDS_HUMAN` 走立法通道让立法者
签字,而不是自己装上去。

→ **这把刀是起草书记员,不是立法者**(人是立法者非操作员)。任何会成为硬不变量、或影响面宽的
候选,默认走 PROPOSE-only。

### ③ 演绎通道的假阳性(演绎通道)

不是每个 guard clause 都是真不变量——有防御性废码、死检查、任务专属检查。

→ **对策**:演绎通道只采不判;靠存活测试与强度分类过滤。过不了存活测试
的(只对某任务成立)判 ◊ 踢下;过不了强度分类的(不是"必须",只是碰巧这么写)丢弃。

## 承重铁律:没有溯源,不许进卡(落卡阶段)

每条落卡的不变量都必须能指回**一个执行点**(通道一)或**一次真实失败**(通道二)。

> **没有溯源的候选,不许进卡。它是猜测,不是恢复出来的规则。**

这是 invariant-extract 区别于"让 LLM 编一堆听起来对的规则"的根本分界。责任卡不是越积越多的
散文,是越积越多的、带溯源、可验证、每条都来自一次真实断裂的规则。

## 边界 / 越权反模式(别把这把刀做成别的刀)

- **别填绑定列**:claim-resolution / 信号绑定与本刀正交(原理稿 §9)。本刀只填不变量列 + 可覆盖默认列。
- **别重做本体**:名词 / 边界 / 统一语言是 dos-extract 的活。本刀消费 `dos.yaml`,不重抽。
- **别生成 done_when**:任务级判据是 acceptance-spec / done-when-pipeline 的活。存活测试把 ◊ 踢给它们。
- **别把领地硬不变量标成 R00x**:R00x 是宪法级;领地硬不变量默认停在责任级,可提案上升、但不自动当宪法。
- **别静默塞进 done_when**:□ 不变量写 `Territory.invariants`(dos 0.1.14 已落地),hard 经人签生效;绝不压进 done_when。

## 一处理论诚实(引 KAOS 时的措辞)

KAOS 的 obstacle 分析**原义**是:从目标**前向演绎**出会阻碍它的障碍,再去消解。本 skill 反过来
用——**从观测到的真实失败,溯因出被违反的不变量**——是对 obstacle 模型的**工程化改造**。

→ 写 decisions.md / 对外解释时,诚实表述为"借 obstacle 模型的精神做溯因恢复",**不要说成
"KAOS 就是这么干的"**。轴(失败是不变量的主信号)是对的;字面机制是我们的改造。
