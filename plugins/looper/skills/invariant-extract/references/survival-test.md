# □/◊ 存活测试 + 强度分类 —— 整把刀的命门

> 这是 存活测试与强度分类的完整判据。invariant-extract 的全部质量,系于这一刀切得准不准。
> 理论根:KAOS 用时序逻辑把两层切开(下),Model Spec 给强度分类的测试(再下)。

---

## 一、为什么这是命门

同一个词"规约",指着两个东西:

- **任务级 = ◊(Achieve)**:`P ⇒ ◊Q` —— "在某个触发下,最终要达到"。这是 `done_when`:
  只对**这一个 Run** 负责,达成即弃。
- **责任级 = □(Maintain / Avoid)**:`□(P→Q)` 与 `□(P→¬Q)` —— "任何时刻都必须保持 / 必须避免"。
  这是**常驻不变量**:**每一个 Run 都必须守**,不随任务变。

把 □/◊ 判错 = 重演 G2 闸门质量崩坏的那个两层混淆。invariant-extract 只该往卡上放 □;
任何 ◊ 混进来,都会让这块领地的常驻法被一次性任务目标污染。

（KAOS 是这把切刀的来源;Maintain/Avoid = □,Achieve = ◊。obstacle 模型则是失败该挂的层。）

---

## 二、存活测试

每个候选不变量,问同一个问题:

> **它能不能活过揭示它的那次 Run?**

操作化:设想这块领地未来一次**与揭示它的那次毫不相关**的 Run,这条规则是否仍**必须**成立。
**这个测试默认相对「这块领地的目的」问**(目的 = name + kpi + dos.scope;见 SKILL.md "目的:贯穿
全程的取景框")。同一条候选,换一块目的不同的领地问,答案可能不同——这正是下面分层探针的钩子。

| 结论 | 判定 | 去向 |
|---|---|---|
| 仍必须成立 | **□** | 留在不变量卡 |
| 只相对某个任务目标才成立 | **◊** | **踢回 done_when 生成器**(acceptance-spec / done-when-pipeline) |

被踢回的 ◊ **不是垃圾**,它是合法的任务级判据,只是放错层了——明确移交,别丢弃。

### 裁断原则:宁可踢下,不可错升

- **漏掉一条 □**(误判成 ◊ 踢下去):代价小——下次失败会**再次抓到**它,棘轮会补回来。
- **错升一条 ◊ 成 □**(塞进卡):代价大——它会**勒死整块领地**,让每个无关的 Run 都被一条
  本不该普适的规则卡住。

所以**边界难判时,默认判 ◊(踢下)**,把"升成 □"的举证责任压在候选身上。

### 几个判别例子

- "退款金额必须经过核验才能写进回答" → 任何一次客服 Run 都得守 → **□**。
- "这次必须把退款额算成 ¥200" → 只对这一个工单成立 → **◊**,踢回 done_when。
- "永不在没有重试预算下调用该外部 API" → 任何 Run 都该守 → **□**。
- "这次要把超时阈值设成 5s" → 任务参数 → **◊**。

## 二点五、分层探针:目的不仅判"是什么不变量",还判"挂哪一层"

存活测试相对**本领地目的**问;再并行做一次**换目的**的思想实验,落点立刻清楚:

| 把候选代入… | 结果 | 落点 |
|---|---|---|
| **哪块领地的目的都成立**(purpose-independent) | 目的换了它不变 | **宪法级(R001–R003)嫌疑** → 上浮、各领地继承 |
| **只在某块领地的目的下成立** | 目的一变它就消失 | **领地级 □** → 留本卡 |
| **只相对某个任务成立** | 任务一变它就消失 | **◊** → 踢回 done_when 生成器 |

> **目的不变它就上浮,目的一变它就消失。**

**诚实的边界:探针给嫌疑,不给判决。** 本刀领地锁定时绑定**恰好一个**领地,**无法独自验证**"代入
所有领地的目的都成立"——它只能把候选**标为 purpose-independent 嫌疑**,作为 `NEEDS_HUMAN` 送立法,
由跨领地比较 / 立法者裁定是否升 R00x。升宪法是立法行为,不是这把刀的权限(起草书记员,不是立法者)。
purpose-independent 的候选**不该**在每块领地里重溯一遍、复制进每张卡——那是把宪法摊薄成 N 份副本。

## 三、强度分类

存活下来的 □,用 Model Spec 的测试再分两栏:

> **「一个任务能不能合法地违反它?」**

| 结论 | 栏位 | 形态 |
|---|---|---|
| **不能**合法违反 | **硬不变量**(root,不可覆盖) | 多为禁止式;`THE SYSTEM SHALL NEVER …` |
| **能**在护栏内合法违反 | **可覆盖默认** | 带 **authority level**,任务级可在护栏内覆盖 |

**注意:能被合法违反的,就不是 □ 硬不变量,别混进硬规则栏。** 它进"可覆盖默认"栏。
`□` 这个标签严格只属于不可违反的那批;可覆盖默认是更软的一层,靠 authority level 治理覆盖冲突
(Model Spec 指挥链:高 authority 覆盖低,同级冲突 default to inaction → 升级立法)。

### 海拔注记(对原理稿 §5 的收口)

原理稿 §5 把硬不变量标成"R00x 类"。**本 skill 收口:领地级抽出的硬不变量默认不标 R00x。**

- **R00x = 宪法级**,跨整个系统,由 dos-extract / 立法产出。
- **本刀的硬不变量 = 责任级**,绑定在一块 Territory 上。
- 它**可以**在落卡阶段*提案上升*为系统 R00x(走立法通道、人签字),但**默认停在责任级**,
  不自动当 R00x。把领地硬不变量直接叫 R00x,就把宪法 / 责任两层糊在一起——这正是本刀反对的。

## 四、和演绎通道假阳性的关系

演绎通道会采进防御性废码、死检查、任务专属检查(失败模式③)。它们多半:

- 过不了**存活测试**(只对某任务成立)→ 被存活测试判 ◊ 踢下;或
- 过不了**强度分类**(根本不是"必须",只是"碰巧这么写")→ 在强度分类落不进任何一栏,丢弃。

所以演绎通道只管"采",判真伪交给这两步——这是设计,不是疏漏。
