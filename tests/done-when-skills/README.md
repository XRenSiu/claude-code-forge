# tests/done-when-skills/

Verification battery for the `done-when-pipeline` plugin, generated by running the plugin's own `test-suite-generator` skill on the meta-spec at `specs/done-when-skills/`.

## Layout

```
tests/done-when-skills/
├── README.md                          (this file)
├── existence.sh                       4-A: 24 file-existence checks
├── unit/
│   └── test_structure_lints.py        4-B: 25 markdown structural lints (REQ-001 .. REQ-016)
├── integration/
│   └── test_cross_plugin.py           4-C: 4 checks against ratchet and persona-distill
└── (4-D / 4-E / 4-F intentionally absent — see below)
```

## What each layer is for, and why some are absent

| Layer | Status | Reason |
|---|---|---|
| 4-A existence | runnable | direct file-presence checks |
| 4-B unit (structure lints) | runnable | the plugin source is markdown; "unit tests" reframed as text-level structural lints |
| 4-C integration (cross-plugin) | runnable | confirms ratchet + persona-distill skills referenced by INTEGRATION.md actually exist and behave as our doc claims |
| 4-D e2e | **deferred** | true e2e = invoke `/acceptance-spec` and `/test-suite-generator` in a fresh Claude Code session. This dogfooding pass runs the source directly, so e2e is out of scope for this round. Recommended after `~/.claude/settings.json` was updated with the plugin enabled — restart the session and inspect the live commands. |
| 4-E mutation | **N/A** | mutation testing requires source code with branching logic. Plugin source is markdown, no AST mutations apply. |
| 4-F fitness | **deferred** | rubric scoring uses the "manual fresh-Claude-session workflow" (see `plugins/done-when-pipeline/skills/test-suite-generator/references/fitness-rubric-guide.md`); not appropriate inside this automated verification round. |

## Run

```bash
# from repo root
bash tests/done-when-skills/existence.sh
python3 tests/done-when-skills/unit/test_structure_lints.py
python3 tests/done-when-skills/integration/test_cross_plugin.py
```

All three should exit 0 on a clean tree. If any fails, consult `specs/done-when-skills/_skill-issues.md` — most likely the failure indicates either a regression in `plugins/done-when-pipeline/` or a newly-discovered design-doc gap that needs another fix-and-loop round.
