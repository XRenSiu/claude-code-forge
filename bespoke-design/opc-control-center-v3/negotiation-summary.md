## 1. Core tone (3 sentences)

OPC Control Center is shaped as **a quiet engineered cockpit for a solo operator running an AI team at 11pm** — not a dashboard SaaS, not an IDE, not a chat client. The visual logic is **dark-native canvas + single chromatic anchor (blue-violet) + everything-else achromatic**, with depth coming from luminance stepping (no shadows on dark) and structure coming from typography weight + spacing rhythm rather than visual chrome. The voice in copy is *engineered, not painted*: imperative short verbs, no exclamation marks, no emoji in chrome, no orchestral marketing tone — designed so the operator's attention budget goes to AI thinking-streams and HITL decisions, not to the interface itself.

| 3 | 4 functional accents (success/warning/error/info) | OPC has 4 distinct event-categories (PRD §6.2); 4-color set is the smallest semantic vocabulary covering them | vercel + github (functional-only, blue-as-interaction-green-as-success) |
| 4 | Translucent borders `rgba(255,255,255,0.06–0.08)` | Structure without visual noise; opaque borders on dark create "hard edge visual walls" | linear-app (translucent-borders rule) |
| 5 | Geometric humanist sans, system-ui stack, weights `[400, 500, 600]` (never 700) | Cold-start render is instant; weight 700 is overstatement against engineered-restraint tone | linear-app + github + mintlify |
| 6 | Display tracking `-0.022em` at 32px+, progressively relaxing to body | Headlines feel "minified-engineered", body stays readable | linear-app + cursor + notion |
| 7 | Card radius **4px** (calibrated subtle-engineering) | 0px = too sharp; 12px+ = too friendly-consumer; 4px is the just-soft-enough sweet spot | notion (subtle-card-radius) |
| 8 | Primary CTA = solid brand pill bg on dark canvas, 6px radius (NOT pill) | Brand color is the highest-contrast attention anchor on dark; pill radius is reserved for status badges only | vercel (dark-pill-cta rule) |
| 9 | Pill status badges (`radius: 9999`) for status/state/role/tier ONLY | Color-equals-meaning instant scan; never decoratively used | github (pill-status-badge) |

| 11 | 8px base spacing scale `[1, 4, 7, 8, 11, 12, 16, ..., 80, 120]` with optical micro-increments | Mathematical precision + optical correction; section padding ≥ 80px to let content breathe | linear-app + vercel + cursor |
| 12 | Depth via luminance stepping, NOT drop shadows on dark | Drop shadows on dark are invisible at low alpha and look like dirt at high alpha | linear-app (luminance-stepping) |
| 13 | Shadow-as-border (`box-shadow: 0 0 0 1px rgba(0,0,0,0.08)`) reserved for rare light overlays | The dark medium uses luminance; light overlays (embedded docs, OS dialogs) borrow vercel's technique | vercel (shadow-as-border) |
| 14 | Voice: imperative short verbs ("Resume", "Halt", "Replace skill"); no exclamation marks; no emoji in chrome | engineered_not_painted + infrastructure_made_invisible | linear-app + vercel (voice rules) |
| 15 | Anti-pattern enforcement: forbidden lists from vercel/linear/mintlify aggregated into §7 Don'ts | Discipline of cool-restraint + no decoration creep + no marketing creep | vercel + linear-app + mintlify (3 anti-pattern rules) |

### The 2 remaining warnings (informational, no v3 output edits needed)

1. **`concretization_of_undefined` is a novel schema label** — used for the Functional Accents `accent_set_size` modification (vercel says "small_set" without cardinality; we picked 4). It transparently sidesteps `out_of_source_range: true` for what is technically a value injection. The decision's confidence is already medium for an unrelated reason, so practical calibration is fine. Future skill schema work should document how `concretization_of_undefined` interacts with confidence-downgrade policy.

2. **Mild kansei coverage circularity** — Functional Accents claims `addressed_elsewhere: [confident → Single chromatic anchor]`, and Single chromatic anchor reciprocally claims `[structured, calm via achromatic chrome]` elsewhere. Coherent in isolation but partly self-referential. Future skill could add a top-level `kansei_primary_addresser` matrix per kansei word so reviewers can verify no kansei word relies solely on bidirectional cross-pointers.

### Production-related warning (not a defect, by design)

- **coherence_check `grid_consistency` warning**: spacing values `[11, 19, 22, 35]` are off the strict 8-base grid. This is **by design** — verbatim from `linear-app-spacing-8px-base-with-optical-008.action.includes_optical_micro_adjustments: true`. Reserved for optical correction, not as base spacing tokens. The check correctly notes them; the rule correctly uses them; the warning is acknowledged-and-accepted.

> "I don't want Ruler as secondary archetype — it feels too authoritarian for a solo-operator tool. Try Magician instead."

> "I want a higher-saturation brand. The current 0.47 saturation looks too washed-out for the brand mark."

> "The 4-tier text scale is too granular — I want 3 tiers."

> "Add `composed` to the design-token level, even if it means a 5th surface step or a multi-lane visualization standard."
