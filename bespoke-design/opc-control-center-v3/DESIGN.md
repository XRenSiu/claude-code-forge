## 1. Visual Theme & Atmosphere

OPC Control Center is **a quiet command surface for someone who decided to run a one-person company by piloting an AI team**. The interface should feel like sitting in a clean, well-lit cockpit at 11pm: every dial is necessary, every readout is honest, nothing is shouting for attention. It is *not* a dashboard SaaS template, *not* a chat client, *not* a developer IDE — it is a control center where the operator's leverage comes from being able to *see* what the AI is thinking and *intervene* at the right moment.

The base canvas is **near-black with an imperceptible cool undertone** (`#0B0D10` range), with luminance-stepped surfaces for cards / panels / modals — depth comes from light, not from drop shadows. A **single chromatic anchor in the blue-violet 240–260° range** carries the brand and marks live agent actions; everything else (text, borders, surfaces) is achromatic. Borders are **semi-transparent white at 0.05–0.08 alpha** — present enough to organize, absent enough to let the event stream breathe.

Typography is **geometric humanist sans with progressively tightening tracking at display sizes** (`-0.022em` at 32px+) and a **three-weight ladder** (400 / 500 / 600 — never 700). The voice in copy is *engineered, not painted*: imperative short verbs ("Resume", "Halt", "Replace skill"), no exclamation marks, no emoji in chrome, no orchestral marketing tone. CTAs are **near-black pills on light overlays and luminance-stepped surfaces on dark** — never gradient, never glowing.

Where this differs from neighboring dev tools:
- **vs. Linear**: Linear is a project tracker; OPC adds *event-stream-as-first-class-citizen*. Tracking density should still match Linear, but typography hierarchy must accommodate a continuously scrolling thinking-stream column (uses `mintlify-three-weight` rule for reading-comfort secondary).
- **vs. Cursor**: Cursor is an IDE; OPC is a *task control center* — cards / kanban / cost ledgers dominate, code panels are secondary.
- **vs. Coze/Dify**: Those use bright SaaS palettes and node-graph ornamentation. OPC explicitly forbids both.

### Key Characteristics

- **Base canvas**: `#0B0D10` near-black, cool undertone, luminance steps `+0.04` per level (canvas → panel → elevated)
- **Brand color**: `HSL 248° 47% 60%` (single anchor, used only on primary CTA / live agent indicator / brand mark)
- **Text scale**: 4-tier achromatic from `#F2F4F7` (primary) → `#62666D` (quaternary), no pure white, no pure black
- **Typography**: geometric humanist sans, weights `[400, 500, 600]`, tracking curve `-0.022em → normal` from 32px → body
- **Spacing base**: 8px with optical micro-increments `[1, 4, 7, 8, 11, 12, ...]`, section padding ≥ 80px
- **Card radius**: `4px` (calibrated subtle-engineering, not friendly-large)
- **Border**: `rgba(255,255,255,0.05–0.08)` translucent on dark; **never** opaque on dark, **never** drop-shadow on dark
- **Voice**: imperative + technical + restrained; no decorative motifs, no exclamation marks, no emoji in chrome
- **Status badges**: pill (`radius: 9999`) with strong color semantics, used **only** for status / state / role / tier — never decoratively
- **CTA hierarchy**: solid primary → translucent (`rgba(255,255,255,0.05)`) secondary → text-only tertiary

## 2. Color Palette & Roles

### Strategy

**Dark-native medium with single chromatic anchor and a strict semantic accent set.** Color carries meaning, never decoration. The control center spends most of its time showing live event streams and cost data, so the eye must be able to distinguish "this is the brand", "this is live agent activity", "this is success / error / warning" within 200ms — and the rest of the chrome must dissolve into the background.

- `code-text`: `#C9CCD2`

## 3. Typography Rules

### Font Family

- **Primary**: `-apple-system, BlinkMacSystemFont, "Inter Variable", "SF Pro Text", system-ui, sans-serif` — geometric humanist sans, OpenType features (`liga`, `lnum`, `locl`) enabled.
- **Mono (code only)**: `ui-monospace, "SF Mono", "JetBrains Mono", Menlo, monospace`. Strict separation from prose.
- **No webfont download** in the dev tool surface; use the system stack so cold-start render is immediate (verbatim adoption of `github-typography-system-ui-only-004` for a control center where load latency hurts perception).

### Hierarchy (size + tracking + weight)

| Role | Size | Weight | Letter-spacing | Use |
|---|---|---|---|---|
| `display-xl` | 48px | 600 | `-0.022em` | Empty-state hero only |
| `display-lg` | 32px | 600 | `-0.022em` | Task detail page title, modal headline |
| `heading-lg` | 24px | 600 | `-0.012em` | Section titles in cards |
| `heading-md` | 18px | 500 | `-0.008em` | Phase name, panel titles |
| `body-strong` | 14px | 500 | `0` | Card titles, button labels |
| `body` | 14px | 400 | `0` | Default copy, event-stream content |
| `meta` | 12px | 400 | `0` | Cost, timestamps, lane quota |
| `mono-sm` | 12px | 400 | `0` | Tool-use args, JSON inline |

### Principles

1. **Tracking curve**: aggressive negative at display, progressively relaxing to normal at body. Verbatim from `linear-app-typography-display-compression-005`: `tracking_curve: scales_with_size_progressively_relaxing`.
2. **Weight ladder is narrow**: weights used are `[400, 500, 600]`. **Never 700** — overstatement breaks the engineered restraint (verbatim `mintlify-typography-three-weight-006.action.no_weight_700_in_body: true`).
3. **Hierarchy priority**: size and tracking carry hierarchy first; weight carries it second (verbatim `mintlify.hierarchy_priority`).
4. **Mono only for code/data**, never for prose, never for chrome labels (verbatim `mintlify.mono_strict_separation: true`).
5. **OpenType features mandatory**: at minimum `liga` for ligatures and `lnum` for tabular figures (cost columns must align).

## 4. Component Stylings

### Buttons

**Primary CTA** — solid, single dominant action per surface:
- bg: `var(--brand)` (HSL `248° 47% 60%`) on dark canvas; on the rare light overlay (e.g. embedded docs viewer), use near-black `#171717`
- text: `#FFFFFF`
- radius: `6px` (verbatim `vercel-components-dark-pill-cta-007.action.primary_cta_radius_px: 6`)
- height: 32px (4-row spacing scale)
- **Never** pill radius on primary action. **Never** gradient. **Never** glowing border.

**Secondary** — translucent, presence without competition:
- bg: `rgba(255, 255, 255, 0.05)` (adapted from `notion-components-translucent-secondary-button-009.secondary_button: rgba(0,0,0,0.05) translucent gray` — inverted for dark canvas)
- text: `var(--text-primary)`
- radius: `6px`
- hover: alpha → `0.08`; active: scale `0.98` (no scale `1.05` exuberance)

**Tertiary** — text-only, no chrome:
- text + underline-on-hover only
- Used for skill-replacement links, lane-config edit, etc.

### Cards (task cards, lane status, artifacts)

- bg: `var(--surface-2)`
- radius: `4px` (verbatim `notion-components-subtle-card-radius-010.action.card_radius_default_px: 4` — the calibrated "minimal_engineering_softness, not friendly-large not sharp-engineering")
- border: `var(--border-default)` (translucent white 0.06)
- padding: `16px 20px` (8px scale, optical adjustment for text-leading area)
- hover: surface step up to `surface-3`, no shadow

### Status Badges

- shape: pill (verbatim `github-components-pill-status-badge-005.action.badge_shape: pill`, `badge_radius_px: 9999`)
- color: strong functional semantics — 4-color functional set (success / warning / error / info) + optional brand color for "live agent" indicator only
- text: 11px / weight 500 / tracking `+0.02em`
- **Use only** for status, state, role, tier (verbatim `github.action.use_for: [status, state, role, tier]`). **Never decoratively.**

### Inputs

- bg: `var(--surface-1)` (recessed below the panel)
- border: `var(--border-default)`, focus → `var(--border-emphasized)` + 1px brand-tinted ring at `rgba(123, 122, 217, 0.20)`
- radius: `6px`
- height: 32px / 40px (compact / default)

### Drop Zones (HITL replacement, skill drag-target, artifact reattach)

- border: **dashed** `1.5px var(--border-emphasized)` (adapted from `stripe-components-dashed-affordance-009.action.dashed_border_use: placeholder_drop_zones`)
- bg: `var(--surface-1)` with translucent brand overlay `rgba(123, 122, 217, 0.04)` on hover
- The dashed→solid transition signals committed state (verbatim `stripe.solid_border_for_committed_state: true`)

### Navigation

- Sidebar: `var(--surface-1)` background, no border, active item indicated by **left edge brand bar** (3px) + text-primary; inactive items at `text-tertiary`
- Top bar: `var(--canvas)` background (no separation from canvas — separation comes from panel below dropping in luminance), 48px tall

### Distinctive: Event Stream Row

The thinking/tool-use stream is a primary surface of this product. It uses:
- bg: `transparent` (sits directly on `surface-1` panel)
- left rail icon: 16px, color matches event-type accent (info/warning/error/etc.)
- collapsed default: 1 line, `text-secondary`, mono for tool args
- expanded: 2-tier indent, monospace JSON syntax-tinted
- **Never** add a card chrome around each event — that becomes visual noise at 50+ events/min

## 5. Layout Principles

### Spacing scale

**8px base with optical micro-increments** (verbatim `linear-app-spacing-8px-base-with-optical-008.action`):

```
base_unit: 8
scale: [1, 4, 7, 8, 11, 12, 16, 19, 20, 22, 24, 28, 32, 35]
includes_optical_micro_adjustments: true
```

The 1 / 7 / 11 / 19 / 22 micro values are reserved for **optical correction** (e.g. compensating for letter-overhang in display headlines) — never as base spacing tokens used by component authors.

For section-level padding: `min: 80px`, `max: 120px` (verbatim `vercel-spacing-jump-scale-008.section_vertical_padding_min: 80`, `_max: 120`).

### Grid / Whitespace philosophy

- **No explicit section dividers** (verbatim `linear-app.no_explicit_section_dividers: true`). Sections separate via spacing rhythm and luminance step alone.
- Whitespace is treated as **gallery emptiness** at section boundaries — generous on purpose, not stingy (informed by `vercel-color-extreme-white-minimalism-001.whitespace_philosophy: gallery_emptiness`, adapted for dark canvas).
- Inside a panel, content can be dense (the operator wants to see status of 5 parallel tasks at a glance) — but section boundaries breathe.

### Border radius

| Element | Radius | Source |
|---|---|---|
| Cards | 4px | `notion.card_radius_default_px: 4` |
| Inputs / Buttons | 6px | `vercel.primary_cta_radius_px: 6` |
| Status badges | 9999 (pill) | `github.badge_radius_px: 9999` |
| Modals | 8px | derived (one step up from card to mark elevated context) |
| App shell corners | 0 | full-bleed dark canvas |

### Container widths

- Task board kanban: full-bleed up to 1920px, then center with margin
- Task detail / settings: max `880px` content column
- Cost ledger / lane config: `1200px` to fit 6-col data table

## 6. Depth & Elevation

### Strategy

**Background luminance stepping, not drop shadows** (verbatim `linear-app-elevation-luminance-stepping-007`).

```
depth_strategy: background_luminance_stepping
no_drop_shadow_on_dark: true
level_increments_via: bg_alpha_white +0.02 per step
use_inset_shadow_for_recessed: rgba(0,0,0,0.2) 0px 0px 12px 0px inset
```

> **Implementation note**: The verbatim rule above says `+0.02 per step`, but the §6 Levels table below uses `+0.04`. This is cross-rule reconciliation: the foundational `linear-app-color-dark-native-001.surface_strategy` (verbatim `luminance stepping (canvas → panel → elevated, +0.04 lightness per step)`) supplies the larger step, which is needed because OPC has 4 visible levels and `+0.02` would make levels 2 and 3 indistinguishable on most non-OLED displays. Trace: see `provenance.yaml#decisions[Luminance stepping for depth].adaptation.modifications[lightness_step_amount]`.

### Levels

| Level | Token | Value | Use |
|---|---|---|---|
| -1 | `surface-recessed` | `inset 0 0 12px rgba(0,0,0,0.2)` | Code blocks, embedded terminal |
| 0 | `canvas` | `#0B0D10` | App background |
| 1 | `surface-1` | canvas + L `+0.04` | Sidebar, panels |
| 2 | `surface-2` | canvas + L `+0.08` | Cards, list rows |
| 3 | `surface-3` | canvas + L `+0.12` | Modals, popovers, dragged cards |
| 4 (rare) | `surface-3` + outline | `+ 1px outline rgba(255,255,255,0.10)` | Topmost popover when stacked over modal |

### Shadow Philosophy

- On the dark surfaces (default state): **never** use `box-shadow` for elevation. Drop shadows on dark are invisible at low alpha and look like dirt at high alpha (verbatim `linear-app.why.avoid: [invisible_dark_shadows_on_dark, plastic_floating_cards]`).
- For the **rare light overlay** (e.g. embedded docs preview, system permission dialog from OS): use `vercel-elevation-shadow-as-border-005` strategy — `box-shadow: 0 0 0 1px rgba(0,0,0,0.08)` (1px spread, zero blur) as a border substitute. This avoids CSS-border clipping at the radius corner.
- For inputs / focused composer: a 1px brand-tinted ring `rgba(123, 122, 217, 0.20)` outside the border, never a shadow blur.

### Motion (depth-related)

- Surface step transitions: `150ms ease-out`
- Modal entrance: scale `0.98 → 1.0` + fade `0 → 1`, `200ms ease-out`. **Never** spring/bounce easing.
- Hover surface-step: `100ms`, no easing curve change

## 7. Do's and Don'ts

### Do

- **Use luminance stepping** for depth — it matches the dark-native medium and reads as engineered rather than skeuomorphic.
- **Reserve the brand color** for primary CTA, live-agent indicator, and brand mark only. Everywhere else, achromatic.
- **Show the event stream collapsed by default** — let the operator opt into detail.
- **Use pill badges** for status / state / role / tier — strong color-meaning mapping, instant scan.
- **Use dashed borders** for placeholder / drop-zone affordance; switch to solid when committed.
- **Use OpenType `lnum`** in any cost / token / quota column so digits align to the same width.
- **Use imperative short verbs** in CTAs: "Resume", "Halt task", "Replace skill", "Approve & continue".
- **Let section spacing carry rhythm**; remove explicit dividers.
- **Use system-ui font stack** so the surface renders instantly on cold start.

### Don't

- **No pure black text** (`#000000`) — use `#F2F4F7` on dark, `#242424` warm-near-black on the rare light surface (verbatim `cal-color-warm-near-black-text-002.no_pure_black_text: true`).
- **No pure white text** (`#FFFFFF`) — cap at lightness `0.97` (`linear-app.max_text_lightness: 0.97`).
- **No weight 700** anywhere (verbatim `mintlify.no_weight_700_in_body: true` + `linear-app.no_weight_above: 590`). Maxes at 600 / 590.
- **No drop shadow on dark surfaces.** (`linear-app.no_drop_shadow_on_dark: true`)
- **No CSS `border` on cards** when shadow-as-border is available — borders clip at the radius.
- **No gradient CTAs.** No glowing borders. (verbatim `vercel.why.avoid: [gradient_ctas]`)
- **No pill radius on primary action** — pill is reserved for badges/status (verbatim `vercel.action.pill_radius_reserved_for: [badge, status_tag]`).
- **No decorative use of brand color** — no brand-colored dividers, no brand-tinted backgrounds outside of focus rings.
- **No marketing orchestral tone**: no "Welcome aboard! 🎉", no "Let's get started!", no exclamation marks in product chrome (verbatim `vercel-voice.avoid_orchestral: true` + `linear-app.avoid_marketing_orchestral: true`).
- **No emoji in chrome** (verbatim `vercel-voice.avoid_emoji_in_voice: true`). Emoji from agent thinking-stream content is fine — it's content, not chrome.
- **No 12px+ card radius** — reads as friendly-consumer, breaks the "engineered restraint" tone (verbatim `notion.why.avoid: [12px_plus_too_friendly]`).
- **No zero-radius cards** — reads as raw / harsh (verbatim `notion.why.avoid: [zero_radius_too_sharp]`).
- **No "warm friendly drift"** — no warm color creep into chrome, no brand color used decoratively (verbatim `linear-app-anti-pattern-warmth-in-chrome-010.action.forbidden`).
  > **Partial-adoption note**: linear's forbidden list literally includes `solid_colored_button_backgrounds`, but we narrow this to "no solid colored button backgrounds **except** the single brand-colored primary CTA per surface" — required for the single-chromatic-anchor decision to have a meaningful expression. Trace: `provenance.yaml#decisions[Anti-pattern enforcement].adaptation.modifications[solid_colored_button_backgrounds]`.
- **No "decoration creep"** — no heavy shadows above alpha 0.1, no decorative use of workflow accents, no inner glow ornament (verbatim `vercel-anti-pattern-decoration-creep-010.action.forbidden`).
- **No "marketing creep"** — no decorative gradient buttons, no heavy box-shadow competing with reading, no mono-typography for prose (verbatim `mintlify-anti-pattern-marketing-creep-010.action.forbidden`).
- **No bright SaaS palette.** No purple gradients. No Inter-default-Tailwind look. (per skill `references/anti-slop-blacklist.md` + product reverse_constraint: `ai_slop`, `generic_corporate`, `enterprisey`, `hypey`)

## 8. Responsive Behavior

OPC Control Center is **desktop / web-first**, explicitly **not** mobile-first per PRD §3.3 (mobile-heavy users are reverse audience). Two breakpoints, both above tablet:

| Breakpoint | Width | Layout |
|---|---|---|
| `compact` | 1024–1439px | Sidebar collapses to icon rail (56px). Kanban shows 4 columns at a time, horizontal scroll for the rest. |
| `default` | 1440–1919px | Sidebar expanded (240px). Kanban shows all 6 columns. |
| `wide` | ≥ 1920px | Sidebar expanded. Kanban + parallel run-view side panel (380px right rail). |

### Below 1024px (laptop screens, secondary)

- Single-column stacked layout; kanban becomes vertical list grouped by status.
- Run view becomes full-width modal sheet rather than side panel.

### Touch targets

Although primary use is desktop with mouse/keyboard, when a touch event is detected (e.g. user opens on iPad), interactive elements raise to 44px minimum height. Status pill stays 24px (read-only).

### Collapsing strategy

- Kanban columns: priority by status — `Running`, `Awaiting Review`, `Failed` always visible; `Done` / `Backlog` collapse first
- Cost ledger columns: priority `Phase`, `Cost`, `Lane`; `Skill`, `Duration`, `Model` collapse below 1280px
- Sidebar: collapses to icon-only at < 1280px viewport

### Image / media behavior

- Artifact previews maintain aspect ratio, max-height 60vh in detail view
- Diagrams in PRD/spec artifacts: full-width, no border (let the surface step provide containment)

### Quick Color Reference

```
surface-1:        #13161A
surface-2:        #1B1E23

```

### Example Component Prompts

**Build the task board kanban column header:**
> Card-less column header at the top of each kanban column. Background `transparent` (sits on `surface-1`). Title in `body-strong` (14px, weight 500, achromatic `text-primary`), followed by a count badge (pill, `surface-2` background, `text-tertiary`, 11px). Below the header: a 1px translucent divider `border-default`. No icons in the header. Total header height 36px including 12px bottom margin to first card.

**Build a task card on the kanban board:**
> Background `surface-2` (`#1B1E23`). Border-radius `4px`. Border `1px solid border-default`. Padding `16px 20px`. Title at `body-strong` (14px / 500), one line truncated. Two metadata rows below at `meta` (12px / 400 / `text-tertiary`): row 1 = phase name + progress (e.g. "Phase 3/5: Drafting"), row 2 = lane chip (pill badge with lane color) + cumulative cost (mono digits, `lnum` OpenType feature, right-aligned). On hover: surface steps to `surface-3`, no shadow. On drag: surface stays `surface-3`, border becomes `border-emphasized`, cursor: grabbing.

**Build the live event-stream row:**
> One row in the run-view event stream. No card chrome — sits directly on the panel's `surface-1`. Layout: 24px left rail with a 16px event-type icon (color matches event accent: info/warning/error/brand for live-agent). Right of rail: one line of content at `meta` (12px), truncated. Tool-use args render in mono. Click to expand: row grows to multi-line with full JSON tinted by mono syntax. Vertical padding 6px collapsed / 12px expanded. No border between rows; rhythm comes from the 6px padding alone.

**Build the HITL checkpoint banner:**
> Sticks to the top of the run view when checkpoint is pending. Background `rgba(229, 160, 79, 0.08)` (warning color at 8% alpha — the only place a tinted background is allowed in chrome). Border-bottom `1px solid rgba(229, 160, 79, 0.20)`. Height 56px. Left: warning icon + text "Awaiting your decision" at `body-strong` `text-primary`. Right: three buttons inline — "Approve & continue" (primary, brand bg), "Modify next step" (secondary, translucent), "Halt" (tertiary, text-only `error` color). 16px gap between buttons. No animation on entrance — appears instantly because the operator might already have lost attention; the abrupt appearance is part of the affordance.

**Build a lane status card in the lane pool:**
> Card on `surface-2`, radius `4px`, padding `16px 20px`. Header: lane name at `body-strong`, status dot (8px round, `success`/`warning`/`error`) + status text inline at `text-secondary`. Body: 2-line meta — line 1 = "Quota: ~60% remaining" with mono digits, line 2 = "Resets in 2h 14min". Footer: link "Configure rules →" at `text-tertiary`, hover → `text-secondary`. Width 280px in the grid; gap 16px between cards.

### Iteration Guide (top 7 rules to remember)

1. **Brand color is rare ammunition.** If you find yourself adding `#7B7AD9` to a divider, a chip, an icon, or a hover background — stop. The single chromatic anchor must stay precious. Use achromatic.
2. **Depth = luminance, not shadow.** Cards step up from the panel below them via `+0.04` lightness. If you reach for `box-shadow`, ask whether you actually meant to use the next surface level.
3. **Tracking tightens at display sizes.** Anything ≥ 24px gets `letter-spacing: -0.012em` minimum; ≥ 32px gets `-0.022em`. Body stays at `0`.
4. **Three weights only**: 400 for body, 500 for emphasis / labels / button text, 600 for headlines. **Never 700.**
5. **Status pill is for status only.** If you're tempted to use it as a decorative chip / generic tag, pick a different component.
6. **Voice = engineered, not painted.** Imperative verbs ("Resume", "Halt"). No exclamation marks. No emoji in chrome.
7. **Section spacing carries rhythm.** Don't add `<hr>` or visual dividers between sections — let `padding: 80–120px` do the work.
