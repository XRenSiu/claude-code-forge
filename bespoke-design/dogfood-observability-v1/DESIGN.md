# Design System — Self-Hosted Observability (concept: "Event log as a newspaper of record")

> Category: Developer tool (self-hosted observability)
> Generated by bespoke-design-system v1.13.0 (mode: auto) on 2026-06-18
> Winner of 3 divergent candidates (taste-critic rank); anchor: mintlify + vercel restraint; productive tension: serif editorial typesetting + warm paper inside a developer tool.

## 1. Visual Theme & Atmosphere

This is observability reframed as **reading**, not monitoring. Where Datadog throws a rainbow wall of widgets, this system typesets your system's events like a serious broadsheet: a warm paper canvas (`#faf7f1`), ink-black text (`#1a1714`), serif display headings, and a single columnar timeline that reads top-to-bottom like a newspaper column rather than a firehose. The bet is that engineers stare at this for hours, and the calm of a well-set page sustains attention far longer than a dark dashboard of competing accents.

The atmosphere is **editorial, authoritative, archival**. Restraint is the whole game: one warm accent (oxblood `#8c2f2a`), hairline rules instead of boxes and shadows, generous vertical rhythm. The log payload — and only the log payload — is set in monospace; everything around it (titles, metadata, prose) is serif/humanist, which is what makes a stream of events feel like a record being kept rather than a console scrolling.

**Key Characteristics:**
- Warm paper canvas `#faf7f1` → panel `#f3eee4` → sunken `#ece5d6` (tone steps, never shadow)
- Serif display (Newsreader) for headings; humanist-sans (iA Writer Quattro) for body; **mono (JetBrains Mono) ONLY for the log payload line**
- Single warm accent — oxblood `#8c2f2a` — for links, active state, primary CTA; never decorative
- Columnar timeline: a single editorial column (max 760px) with a hairline column-rule and newspaper-style marginal timestamps
- Hairline borders `#c8bfae`; near-zero radius (2–3px); no drop shadows
- Narrow weight ladder (400 / 500 / 600); hierarchy from serif/size/rule, not weight or color

## 2. Color Palette & Roles

### Background (warm paper)
- **Canvas** `#faf7f1` — page background
- **Panel** `#f3eee4` — cards, side rail
- **Sunken** `#ece5d6` — code/log payload background, input wells

### Text (ink)
- **Primary** `#1a1714` (13.6:1 on canvas — exceeds WCAG AAA)
- **Secondary** `#3a352e` — body prose
- **Tertiary** `#6b6357` — metadata, timestamps
- **Quaternary** `#9c9384` — disabled, faint marginalia

### Accent & State
- **Oxblood** `#8c2f2a` — the only chromatic anchor: links, active row, primary CTA. Hover `#a85a4f`.
- **Status** (functional only, all warm-keyed to stay in palette): ok `#2f5c4a`, warn `#9a6d1f`, alert `#7a2e2e`.

### Strategy
- Palette type: warm monochrome paper + single oxblood anchor + warm-keyed status.
- Depth by tone step (paper → panel → sunken), never by shadow.
- Body contrast 13.6:1.

## 3. Typography Rules

### Font Family
- **Display**: `Newsreader` (serif) — event/section headings, the signature move.
- **Body**: `iA Writer Quattro` (humanist sans) — prose, labels, UI.
- **Mono**: `JetBrains Mono` — **log payload lines and raw values only.**

### Hierarchy
| Role | Font | Size | Weight | LH | Notes |
|------|------|------|--------|----|-------|
| Display | Newsreader | 42px | 500 | 1.15 | Page/section title — serif, the identity carrier |
| H1 | Newsreader | 31px | 500 | 1.2 | Stream / service heading |
| H2 | Newsreader | 24px | 500 | 1.25 | Group heading |
| Lede | iA Writer Quattro | 19px | 400 | 1.5 | Section intro prose |
| Body | iA Writer Quattro | 16px | 400 | 1.55 | Standard reading |
| Meta | iA Writer Quattro | 13px | 500 | 1.4 | Timestamps, marginalia |
| Log payload | JetBrains Mono | 13px | 400 | 1.5 | The actual log/trace line — mono ONLY here |

### Principles
- **Serif headings are non-negotiable** — the moment headings flip to a sans/Inter, the entire "newspaper of record" identity collapses into a generic dev tool.
- **Mono is quarantined to the payload.** Never set titles, labels, or prose in mono.
- Hierarchy comes from serif-vs-sans + size + hairline rules, not from weight (max 600) or color.

## 4. Component Stylings

### Buttons
- **Primary CTA**: oxblood `#8c2f2a` bg, `#faf7f1` text, 2px radius, 8×16 padding, hover `#a85a4f`.
- **Ghost**: transparent, `1px solid #c8bfae`, ink text, 2px radius.

### Cards / containers
- `#f3eee4` panel, `1px solid #c8bfae` hairline, 3px radius, **no shadow**.

### The event row (signature component)
- A row in the columnar timeline: left **hairline column-rule**, marginal timestamp in 13px sans tertiary, severity dot, then the message. The raw payload expands inline in JetBrains Mono on `#ece5d6`.

### Inputs
- `#ece5d6` well, `1px solid #c8bfae`, 2px radius, ink text. Search is a single wide field, no chrome.

## 5. Layout Principles

- **Single editorial column**, max content width **760px** — readability over dashboard sprawl.
- **Marginal timestamp rail**: timestamps sit in the left margin like a newspaper's dateline, separated by a hairline column-rule.
- Spacing base 8px; scale 4/8/12/16/24/40/64/96; section vertical padding ≥ 80px (generous, mintlify-derived).
- Whitespace is the layout: paper breathing room, not dividers, separates regions.
- Radius scale: 2px (controls) / 3px (cards). No pills.

## 6. Depth & Elevation
| Level | Treatment | Use |
|-------|-----------|-----|
| 0 | `#faf7f1` flat | Page |
| 1 | `#f3eee4` tone step + hairline | Cards, rail |
| 2 | `#ece5d6` tone step | Log payload wells, inputs |
| Focus | `0 0 0 2px rgba(140,47,42,0.35)` oxblood ring | Keyboard focus |

**Philosophy**: depth is a tone step on warm paper, never a drop shadow. The page should look printed.

## 7. Do's and Don'ts

### Do
- Set headings in Newsreader serif; keep mono strictly to the log payload.
- Build on warm paper `#faf7f1` with ink `#1a1714`.
- Reserve oxblood `#8c2f2a` for links/active/CTA only.
- Use hairline rules and tone steps for structure; keep one editorial column ≤ 760px.

### Don't
- Don't use Inter/sans for headings — that's the generic-dev-tool collapse.
- Don't add a dark mode "because dev tools have dark mode" without re-justifying against the reading thesis.
- Don't use drop shadows, gradients, or rounded pill cards (the Datadog/enterprise-SaaS look this rejects).
- Don't let any color other than oxblood + functional status enter the chrome.

## 8. Responsive Behavior
- Column stays ≤ 760px; on wide screens, center it and let paper margins grow (don't widen the measure).
- Marginal timestamp rail collapses above the message on < 640px.
- Touch targets ≥ 40px; serif display steps down 42→31→24 across breakpoints.

## 9. Agent Prompt Guide

### Quick Color Reference
- Canvas `#faf7f1` / Panel `#f3eee4` / Sunken `#ece5d6`
- Ink `#1a1714` / Secondary `#3a352e` / Tertiary `#6b6357`
- Accent oxblood `#8c2f2a` (hover `#a85a4f`) / Hairline `#c8bfae`
- Status ok `#2f5c4a` / warn `#9a6d1f` / alert `#7a2e2e`

### Example Component Prompts
- "Build an event row: left hairline column-rule (#c8bfae), marginal timestamp in iA Writer Quattro 13px #6b6357, a severity dot, then the message in 16px. Expanded payload renders in JetBrains Mono 13px on a #ece5d6 well. No card border, no shadow — separation is the hairline rule only."
- "Section heading: Newsreader serif 31px weight 500 ink #1a1714 on #faf7f1, with a thin oxblood underline rule on the active section."
- "Primary CTA: oxblood #8c2f2a bg, #faf7f1 text, 2px radius, focus ring 0 0 0 2px rgba(140,47,42,0.35)."

### Iteration Guide
1. Headings = Newsreader serif. If you reach for Inter, stop — you're collapsing the identity.
2. Mono only on the payload line. Everything else serif/humanist.
3. Warm paper + ink + one oxblood. No second accent.
4. Depth = tone step, never shadow. Radius ≤ 3px, no pills.
5. One column ≤ 760px; timestamps in the margin.
